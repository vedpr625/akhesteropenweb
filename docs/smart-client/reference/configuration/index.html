Configuration Reference

smart.epic.*​

PropertyTypeDefaultRequiredDescription
smart.epic.client-idString—✓Epic application client ID from App Orchard
smart.epic.redirect-uriString—✓Callback URL — must match App Orchard registration
smart.epic.scopesStringlaunch openid fhirUser patient/Patient.rsSpace-separated SMART scopes
smart.epic.launch-urlString—EHR launchOverride the default Epic FHIR base URL
smart.epic.token-url-overrideString—Override the token endpoint (testing only)

smart.fhir.*​

PropertyTypeDefaultDescription
smart.fhir.connect-timeout-msint10000TCP connect timeout for HAPI FHIR client
smart.fhir.socket-timeout-msint30000Socket read timeout for HAPI FHIR client

smart.cache.*​

PropertyTypeDefaultDescription
smart.cache.discovery-ttl-minutesint10SMART configuration cache TTL per ISS
smart.cache.jwks-ttl-hoursint1JWKS cache TTL per issuer

smart.security.*​

PropertyTypeDefaultDescription
smart.security.token-refresh-buffer-secondsint120Seconds before expiry to trigger proactive refresh
smart.security.session-timeout-secondsint3600HTTP session timeout

Full example​

application.yml
smart:
  epic:
    client-id:     ${SMART_CLIENT_ID}
    redirect-uri:  ${SMART_REDIRECT_URI:http://localhost:8080/callback}
    scopes: >
      launch openid fhirUser
      patient/Patient.rs
      patient/Condition.rs
      patient/MedicationRequest.rs

  fhir:
    connect-timeout-ms: 10000
    socket-timeout-ms:  30000

  cache:
    discovery-ttl-minutes: 10
    jwks-ttl-hours:        1

  security:
    token-refresh-buffer-seconds: 120
    session-timeout-seconds:      3600

spring:
  session:
    store-type: redis
    timeout:    3600s
